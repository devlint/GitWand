# v3.1.0 — Fix « Launch Claude Code » + Terminal PTY à onglets

> Design doc · 2026-06-23 · GitWand
> Roadmap: `ROADMAP.md` → "v3.1.0 — Terminal tabs & AI workspace"

## Contexte & objectif

La roadmap v3.1.0 regroupe trois chantiers de tailles très différentes :

1. **Fix prérequis** — l'action "Launch Claude Code" (Agent Sessions, v2.8.0) est câblée mais ne produit aucune session utilisable.
2. **Terminal à onglets** — un terminal intégré avec plusieurs sessions shell simultanées.
3. **AI workspace** — vision exploratoire ("New AI task", diff live de l'agent), explicitement façonnée par les retours utilisateurs.

**Ce document couvre les chantiers 1 et 2 uniquement.** Le chantier 3 (AI workspace) reste exploratoire et fera l'objet d'une spec ultérieure.

### Pourquoi le fix est nécessaire

`agent_session_launch` (`apps/desktop/src-tauri/src/commands/ops.rs:2410-2428`) lance le CLI `claude` avec `stdin`/`stdout`/`stderr` redirigés vers `null` et sans fenêtre terminal — or Claude Code est un TUI interactif qui exige un TTY. Le process meurt immédiatement / tourne invisible : l'utilisateur clique, rien ne se passe. De plus, le frontend (`AgentSessionsPanel.vue:74-78`) pose `active = true` de façon optimiste après un `setTimeout` de 1,5 s, sans confirmation que l'agent a démarré.

### Décisions de cadrage (validées)

| Décision | Choix retenu |
|---|---|
| Périmètre de cette spec | Fix prérequis **+** terminal à onglets |
| Techno terminal | **PTY réel** : `portable-pty` (Rust) + `xterm.js` (front) |
| Sort du `GitTerminal` actuel | **Remplacé** par le terminal PTY |
| Cible de "Launch Claude Code" | **Onglet PTY intégré** (Cursor/Windsurf restent GUI externe) |
| Cycle de vie des onglets | **Scopés au repo actif**, PTY vivants en arrière-plan |
| Refresh auto repo | **Debounced sur idle PTY** (~800 ms) |

## État de l'existant (référence)

- **`GitTerminal.vue`** — n'est PAS un vrai terminal : un *runner de commandes git* (une commande → `git_exec` → affichage). Pas de PTY, pas de shell interactif. Monté en overlay dans `App.vue:2709` (`defineAsyncComponent`, `App.vue:58`). Émet `@refresh` sur les commandes mutantes.
- **`git_exec` / `git_autocomplete`** (`ops.rs:2522-2561`) — exécutés en thread bloquant, renvoient `TerminalResult { stdout, stderr, exit_code }`. Conservés (utilisés ailleurs) mais ne sont plus la base du terminal.
- **`claude_cli_login`** (`ai.rs:652-716`) — contient le pattern cross-platform de lancement d'un terminal *externe* (osascript/Terminal.app · `cmd /k start` · gnome-terminal/konsole/kitty/…). Référence pour le PATH/env, mais **non réutilisé** ici car on lance dans un PTY intégré, pas un terminal externe.
- **`hidden_cmd`** (`git/cmd.rs:245-290`) — enrichit le PATH (Homebrew/npm global), dé-pollue l'env AppImage, propage `GH_TOKEN`/`GITHUB_TOKEN`. On réutilise **la logique PATH/AppImage** pour le spawn du shell, mais **pas** la propagation de tokens (cf. sécurité).
- **`agent_session_list` / `agent_session_launch`** (`ops.rs:2348-2428`) — détection des worktrees outillés (`.claude`/`.cursor`/`.windsurf`/`.mcp.json`), `tool` est une `String` ("claude"|"cursor"|"windsurf"|"other"), `active` calculé via `lsof`/`/proc`.
- **`useRepoTabs.ts`** — onglets multi-repos, identité stable = `path` (les `id` numériques sont régénérés à chaque chargement). Modèle à suivre pour scoper les onglets terminal par `repoPath`.
- **Raccourcis** (`App.vue:2032-2071`) — `⌘T` = nouvel onglet repo (open folder), `⌘W` = ferme l'onglet repo actif, `⌘⇧N` = quick-create worktree, `⌘⇧,` = quick stash, `⌘⇧L` = command log. **`⌘T`/`⌘W` sont déjà pris** → routage par focus nécessaire.

## Architecture

### Vue d'ensemble du flux

```
xterm.js (front)  ──keystrokes──▶  terminal_write(id, data)  ──▶  PTY master write
PTY master read  ──reader thread──▶  Channel<String>  ──▶  xterm.write(chunk) (front)
container resize  ──fit addon──▶  terminal_resize(id, cols, rows)  ──▶  PTY winsize
```

### Backend Rust

**Nouvelle dépendance** : `portable-pty` (crate wezterm). Mesurer l'impact compile + binaire (invariant perf `apps/desktop/CLAUDE.md`) et le consigner dans la PR.

**Nouveau module** : `apps/desktop/src-tauri/src/commands/terminal.rs`.

**Registre de sessions** :

```rust
struct PtyHandle {
    master: Box<dyn MasterPty + Send>,
    writer: Box<dyn Write + Send>,
    child: Box<dyn Child + Send + Sync>,
    // le thread lecteur est détaché ; il s'arrête quand le PTY est fermé
}

static SESSIONS: Lazy<Mutex<HashMap<u64, PtyHandle>>> = ...;
static NEXT_ID: AtomicU64 = AtomicU64::new(1);
```

**Commandes Tauri** (toutes avec wrapper typé dans `backend.ts`, même PR) :

| Commande | Signature | Rôle |
|---|---|---|
| `terminal_open` | `(cwd: String, shell: Option<String>, cols: u16, rows: u16, on_output: Channel<String>) -> Result<u64>` | Ouvre un PTY, spawn le shell, démarre le thread lecteur qui pousse les chunks dans `on_output`. Renvoie le `session_id`. |
| `terminal_write` | `(id: u64, data: String) -> Result<()>` | Écrit dans le PTY master (keystrokes). |
| `terminal_resize` | `(id: u64, cols: u16, rows: u16) -> Result<()>` | Ajuste la taille du PTY (`PtySize`). |
| `terminal_close` | `(id: u64) -> Result<()>` | Tue le child, retire la session du registre (le thread lecteur sort sur EOF). |

**Choix d'implémentation** :
- **Streaming output** via `tauri::ipc::Channel<String>` (et non des events globaux) : ciblé par session, haut débit. Le thread lecteur lit par chunks (buffer 4–8 Ko), décode UTF-8 de façon tolérante (`String::from_utf8_lossy`) et `channel.send(chunk)`.
- **Shell** : ordre de résolution = `shell` (override Settings) → `$SHELL -l` (Unix) → `/bin/bash` ; sur Windows = `%ComSpec%` ou `powershell.exe`. Login shell (`-l`) pour charger le profil utilisateur (PATH, aliases, etc.).
- **cwd** : **validé par `safe_repo_path()`** avant le spawn (règle AGENTS.md). Jamais d'interpolation shell — `CommandBuilder` avec args discrets.
- **Env** : base = env utilisateur + enrichissement PATH (Homebrew/npm global, dé-pollution AppImage) repris de `hidden_cmd`. **On n'injecte PAS** les tokens GitWand (`GH_TOKEN`/`GITHUB_TOKEN`) — un shell interactif que l'utilisateur possède charge ses propres credentials via son profil ; injecter des secrets GitWand dans un shell visible viole "Never log secrets" (les tokens apparaîtraient dans l'env du shell).
- **Cleanup global** : à la fermeture de l'app (ou `WindowEvent::Destroyed`), tuer tous les childs du registre pour ne pas laisser de PTY orphelins.

### Frontend

**Nouvelles dépendances npm** : `@xterm/xterm` + `@xterm/addon-fit`, **lazy-importées** (le panneau est `defineAsyncComponent`) — conforme à l'invariant perf "tout panel en `v-if` défaut-faux est lazy".

**Composable** : `apps/desktop/src/composables/useTerminalSessions.ts`

État (au niveau module, pattern store-composable) :

```ts
interface TerminalTab {
  id: number;            // id local (mémoire), comme RepoTab
  sessionId: number;     // id PTY backend
  title: string;         // titre affiché
  titleManual: boolean;  // true si renommé à la main (n'écrase plus sur onTitleChange)
  alive: boolean;        // PTY vivant ?
}
// scopé par repo : Map<repoPath, TerminalTab[]> + activeTabId par repoPath
```

- Les instances `Terminal` xterm + `FitAddon` sont stockées dans une **map NON réactive** (`Map<sessionId, { term, fit, channel }>`) — jamais dans un `ref`/`reactive` (xterm gère son propre DOM ; le rendre réactif tue les perfs).
- API : `openTab(repoPath, cwd)`, `closeTab(repoPath, tabId)`, `setActive(repoPath, tabId)`, `renameTab(repoPath, tabId, title)`, `tabsFor(repoPath)`, `disposeRepo(repoPath)` (quand un onglet repo est fermé pour de bon).
- **Scope** : changer de repo actif → on affiche `tabsFor(activeRepoPath)` ; les PTY des autres repos **restent vivants** (on ne `terminal_close` que sur fermeture explicite d'onglet, fermeture définitive du repo, ou quit app).
- **Titre auto** : `term.onTitleChange(t => { if (!tab.titleManual) tab.title = t })` — les shells émettent un OSC 0/2 avec la commande/cwd courant. Fallback initial = nom du shell.
- **Refresh auto** : sur réception d'un chunk d'output, (ré)armer un timer debounce 800 ms ; à l'expiration, si l'onglet appartient au repo actif, appeler `repoRefresh()`. Pas de `setInterval` permanent (conforme perf).

**Composant** : `apps/desktop/src/components/TerminalPanel.vue`

- Ancré **en bas**, hauteur **redimensionnable** (drag-handle en haut du panneau, hauteur persistée en `localStorage`).
- Barre d'onglets : titre par onglet, bouton `+` (nouvel onglet), `×` par onglet, **double-clic = renommer** (input inline → `renameTab`, pose `titleManual`).
- Zone terminal : un `<div>` hôte par onglet ; on `term.open(el)` au mount, `fit()` au resize (ResizeObserver) → `terminalResize`. L'onglet inactif est `display:none` (xterm conserve son buffer), pas démonté.
- Remplace l'overlay `GitTerminal` : `App.vue:2709` rend `TerminalPanel`, `App.vue:58` import retiré, `GitTerminal.vue` supprimé. `@refresh="repoRefresh()"` n'est plus câblé via le composant (le refresh passe par le debounce du composable).

### Lancement de session agent (réconciliation)

Dans `AgentSessionsPanel.vue` (et son composable) :

- Bouton **Launch** pour `tool ∈ {claude, other}` → `openTab(activeRepoPath, session.path)` puis `terminalWrite(sessionId, "claude\n")`, ouvre/affiche le `TerminalPanel`, focus l'onglet.
- Pour `tool ∈ {cursor, windsurf}` → ouverture **GUI externe** dédiée (ces éditeurs ne sont pas des CLI TUI ; on ouvre l'app sur le worktree). C'est le point "Make Cursor/Windsurf launchable too" de la roadmap.
- **Suppression du faux `active`** : retirer le `setTimeout` 1,5 s de `AgentSessionsPanel.vue:74-78`. L'état réel vient (a) du PTY vivant pour les sessions lancées en interne, (b) d'un **re-poll `agent_session_list` après lancement** pour confirmer.

### Raccourcis clavier (routage par focus)

`App.vue` `onKeyDown` : introduire un flag `terminalFocused` (posé/retiré par `TerminalPanel` au focus/blur de la zone xterm).

- **Terminal focalisé** : `⌘T` → `openTab` · `⌘W` → `closeTab` (onglet terminal actif) · `⌘1..9` → `setActive` (onglet N).
- **Hors focus terminal** : comportement actuel inchangé (`⌘T`/`⌘W` = onglets repo).

`⌘W` quand le terminal a un seul onglet et qu'on le ferme → masque le panneau (ne ferme pas l'onglet repo).

### Settings (synchroniser les deux interfaces)

Ajouter dans **`useSettings.ts` (`AppSettings`) ET `SettingsPanel.vue` (`Settings`)** — même commit (règle AGENTS.md) :

- `terminalFontSize: number` — défaut `13`.
- `terminalShell: string` — override optionnel du shell (vide = auto-détection). Passé à `terminal_open`.

## Sécurité

- `cwd` → `safe_repo_path()` systématiquement.
- Aucun string-interpolation dans le spawn (CommandBuilder + args discrets).
- Pas d'injection de tokens GitWand dans l'env du shell (cf. ci-dessus).
- `terminal_write` transmet des octets bruts au PTY : c'est l'usage normal d'un terminal (l'utilisateur tape ce qu'il veut). Pas de surface d'injection côté GitWand tant qu'on ne construit pas de commande à partir d'input.

## Performance

- `portable-pty` : mesurer impact compile + taille binaire, consigner dans la PR.
- `@xterm/xterm` (> 50 Ko gzip) : **lazy import obligatoire** (panneau `defineAsyncComponent`).
- Streaming via `Channel` par session (pas d'events globaux qui réveilleraient toute l'app).
- Refresh repo : debounce 800 ms, **aucun `setInterval` permanent**.
- Instances xterm hors-réactivité Vue.

## dev:web parity

`apps/desktop/dev-server.mjs` doit exposer des routes **fonctionnelles** (non-stub) pour `terminal_open`/`write`/`resize`/`close` (mémoire projet : parité dev:web requise pour toute commande Tauri). Comme un vrai PTY est lourd côté Node, l'objectif minimal est un **PTY/echo Node** (via `child_process` avec un shell en mode pipe, ou un simple echo line-based) suffisant pour exercer l'UI du panneau et le composable sans Rust. Le streaming `Channel` étant spécifique Tauri, le mock front utilisera un transport équivalent (event/SSE) côté dev-server.

## i18n

Toutes les strings user-visibles (titres d'onglets par défaut, tooltips `+`/`×`/renommer, libellés Settings terminal, messages d'erreur de spawn) → clés ajoutées dans les **5 locales** (`en`, `fr`, `es`, `pt-BR`, `zh-CN`).

## Tests

- **Vitest — `useTerminalSessions`** : ouverture/fermeture d'onglets ; scope par `repoPath` (les onglets d'un repo n'apparaissent pas pour un autre) ; PTY vivants au changement de repo ; `disposeRepo` ferme bien les sessions ; titre auto vs `titleManual` ; debounce refresh (avec timers fake).
- **Vitest — routage raccourcis** : `⌘T`/`⌘W`/`⌘1..9` agissent sur les onglets terminal quand `terminalFocused`, sur les onglets repo sinon.
- **Manuel desktop** : lancer `claude` depuis Agent Sessions ouvre un onglet utilisable ; `vim`/`htop` fonctionnent (vrai TTY) ; resize ; multi-onglets ; pas de PTY orphelin après quit ; Cursor/Windsurf ouvrent en GUI.
- Pas de mock de la couche git ; pas de régression parité Rust↔JS (le module terminal ne touche pas la résolution de conflits).
