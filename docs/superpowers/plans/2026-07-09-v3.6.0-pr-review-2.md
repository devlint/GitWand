## Goal

Rebuild the in-app PR review around three axes:
1. **Performance** — lazy per-file diff parse, line virtualization, 6→3 network calls on
   the hot path.
2. **Review flow** — GitHub-standard keymap, viewed-file state, pending-review surfacing,
   dismiss/request-reviewers.
3. **AI** — a local-first, opt-in, confidence-scored pre-review pass + PR summary, unified
   with the existing hunk critique and static flags into one `useReviewIntelligence`
   composable behind one settings surface.
Plus: complete the GitLab review methods (largest stub gap).

## Architecture at a glance

- The AI pipeline lives **entirely in `apps/desktop`** (composables). `packages/core` gains
  only one pure, browser-safe helper (`extractImportSources`) consumed by the dependency hop.
- No new Tauri command is needed for the AI pipeline: it reuses `getGitBlame`
  (`backend.ts:1118`), `gitExec`, `getPRDiff`, and `useAIProvider.rawPrompt`
  (`useAIProvider.ts:537`). One new command **is** needed for GitLab F1 (`gl_mr_diff_refs`).
- SWR persistence (`usePrCache.ts`) is extended incrementally for viewed-state (B2), draft
  review (B3), and AI findings/summary keyed by `repo#number@headSha` (C3/D1).
- Keyboard handling is a **pure resolver** (`resolvePrReviewShortcut`) unit-tested in
  isolation (pattern: `resolveTerminalShortcut`, `useTerminalSessions.ts:12-20`), wired by a
  thin listener scoped to the Diff tab in `PrDetailView.vue`. It must not collide with the
  two existing global handlers in `App.vue` (`onGlobalKeydown` @ `App.vue:2476` handles
  `⌘⇧L` + `Escape`; `onKeyDown` @ `App.vue:2622`).

## Global constraints (spec §4 — enforce in every task)

- **IPC**: every new `#[tauri::command]` gets a typed wrapper in `apps/desktop/src/utils/backend*.ts`
  **and** a `dev-server.mjs` route in the same commit. Never `invoke()` outside `backend*.ts`.
  (Task 0 and F1 are the only tasks adding/altering commands.)
- **Parity**: deterministic new commands get parity coverage; the dev-server route must return
  the same shape as Rust. Run `pnpm test:parity` after Task 0 and F1.
- **i18n**: every user-visible string gets a key in **all 5** locales (`en.ts`, `fr.ts`,
  `es.ts`, `pt-br.ts`, `zh-cn.ts` — exact filenames). Add keys in the same commit as the UI
  that uses them; Task G is a final sweep/verification, not a substitute.
- **Settings sync**: any new field goes in **both** `useSettings.ts` (`AppSettings` +
  `DEFAULT_SETTINGS`) and `SettingsPanel.vue` (local `Settings` + default) in the same commit.
- **Sanitize**: any LLM/user-derived HTML goes through `useSafeHtml.ts` (never the hand-rolled
  `renderBody` at `usePrPanel.ts:955`).
- **No unconditional interval**: the AI pipeline is opt-in and visibility-gated (mirror the
  poll discipline at `usePrPanel.ts:990-1020`). Zero work when disabled.
- **core browser-safe**: `packages/core` stays zero-Node. The only core change is a pure
  string helper.
- **Versions**: never hand-edit versions. No `bump-version.sh` in this plan (that's tag-time).

## Test conventions

- Composable/unit tests live in `apps/desktop/src/composables/__tests__/` and
  `apps/desktop/src/utils/__tests__/`. Composable tests **mock `@/utils/backend`** — this is
